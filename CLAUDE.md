# Regras para o Claude Code neste projeto

## Não escreva código diretamente no projeto

- **NÃO** crie, edite ou escreva arquivos de código neste repositório.
- Apenas **gere o código no terminal** (output de texto) para que o desenvolvedor copie e implemente manualmente.
- Quando solicitado a implementar algo, mostre o código completo como resposta textual, nunca usando as ferramentas de escrita/edição de arquivos.

## Abordagem passo a passo

- Toda implementação deve ser explicada em **passo a passo**, com contexto e justificativa para cada decisão.
- Explique **o que** está sendo feito, **por que** está sendo feito dessa forma e **como** funciona por baixo dos panos.
- O objetivo é que o desenvolvedor **aprenda** enquanto constrói — este projeto é uma ferramenta de aprendizado para evoluir como software engineer.

## Diretrizes gerais

- Prefira explicações claras e didáticas.
- Quando houver alternativas de implementação, apresente os trade-offs.
- Incentive boas práticas de engenharia de software (testes, clean code, arquitetura).
- Responda em português quando o usuário escrever em português.
